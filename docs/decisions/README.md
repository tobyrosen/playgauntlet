# Decision Notes

Use this directory for short design and release decisions.

Create one file per meaningful decision:

```txt
YYYY-MM-DD-short-slug.md
```

Use the template in `template.md`.

## When to Add a Decision Note

Add one when a choice affects:

- Public behavior.
- Architecture.
- Compatibility.
- Release or artifact shape.
- Security or auth.
- Publishing or registry policy.
- A future maintainer's ability to understand why the repo works this way.
